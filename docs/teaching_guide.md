# 教学实施与协作指南

## 一、项目简介（发给学生）

《深度学习算法可视化实验平台》是一个本地部署的教学型开源项目。学生通过浏览器上传图片或视频，选择算法运行并查看可视化结果。项目强调模块化协作与可二次开发，适合课程实训与小组项目管理。

## 二、平台功能模块说明

1. 用户登录与退出
2. 算法首页展示
3. 图像上传与预览
4. 视频上传与预览
5. 算法选择与运行
6. 结果展示与对比
7. 历史测试记录
8. 系统日志
9. 算法说明页面
10. 本地部署与测试文档

## 三、算法模块教学意义

- YOLO 目标检测：理解检测任务输入输出、边界框概念
- 图像分割：理解像素级预测与掩码表示
- MNIST 分类：理解分类模型与 softmax 输出
- 超分辨率：理解图像增强与重建思路
- 动作识别：理解视频帧处理和时序特征
- 纹理风格化：理解风格迁移/图像变换的视觉效果

## 四、岗位分工建议（6 人组）

- 项目经理 1 人
- 前端工程师 1 人
- 后端工程师 1 人
- 算法工程师 1 人
- 测试工程师 1 人
- 运维/文档工程师 1 人

## 五、六个岗位详细任务

### 1）项目经理

- 拆解迭代任务，维护看板
- 组织站会与里程碑检查
- 负责合并策略和提交质量把关

### 2）前端开发工程师

- 完成 7 个页面的 UI 与交互
- 接入接口并处理上传、结果展示
- 增强可视化效果和可用性

### 3）后端开发工程师

- 实现 Flask RESTful API
- 处理认证、文件上传、历史和日志
- 维护 SQLite 表结构与查询接口

### 4）算法工程师

- 维护 `models/` 下六个算法模块
- 增强推理效果与运行速度
- 维护算法说明与参数示例

### 5）测试工程师

- 编写接口自动化测试
- 设计冒烟测试与回归测试用例
- 跟踪缺陷与复现步骤

### 6）运维/文档工程师

- 维护部署脚本与运行文档
- 记录版本变更与问题排查
- 组织演示材料与课堂答辩文档

## 六、GitHub 协作开发建议

- 主分支：`main`（稳定可发布）
- 开发分支：`develop`（日常集成）
- 功能分支：`feature/<模块>-<姓名>`（个人开发）
- 小组建议分支：`group1-develop`~`group5-develop`
- 提交流程：feature → group develop → develop → main

## 七、学生提交规范

### Commit Message 规范

- `feat: 新增图片上传预览`
- `fix: 修复视频上传格式校验`
- `docs: 更新部署文档`
- `test: 增加 yolo 接口测试`

### 分支命名规范

- `feature/login-zhangsan`
- `feature/mnist-lisi`
- `fix/upload-bug-wangwu`

### Pull Request 要求

- 标题：`[模块] 功能说明`
- 描述：改动点、测试结果、截图
- 必须关联 issue 或任务编号

## 十、常见报错与排查建议

- 登录失败：检查默认账号和数据库初始化
- 上传失败：检查文件格式和大小限制
- 推理报错：检查输入路径是否存在
- 视频无法播放：检查浏览器是否支持编码
- 页面空白：检查后端是否启动、接口返回是否 200
